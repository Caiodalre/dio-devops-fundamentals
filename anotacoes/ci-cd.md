# CI/CD

## O que significa CI/CD?

CI/CD significa Integração Contínua e Entrega Contínua.

Em alguns contextos, CD também pode representar Continuous Deployment, ou Implantação Contínua.

## CI - Continuous Integration

Integração Contínua é a prática de integrar alterações de código com frequência em um repositório compartilhado.

Normalmente, esse processo envolve:

- Commit do código
- Build da aplicação
- Execução de testes automatizados
- Validações de qualidade
- Identificação rápida de erros

## CD - Continuous Delivery

Entrega Contínua é a prática de manter o software sempre pronto para ser implantado em um ambiente.

Após passar pela Integração Contínua, o software pode ser empacotado e preparado para deploy.

## Continuous Deployment

Continuous Deployment é a implantação automática das alterações aprovadas no pipeline diretamente em produção.

## Pipeline

Pipeline é uma sequência de etapas automatizadas no desenvolvimento e implantação de software.

Um pipeline pode conter:

1. Commit
2. Build
3. Testes
4. Análise de qualidade
5. Empacotamento
6. Deploy

## Objetivo do CI/CD

O objetivo do CI/CD é tornar a integração de código mais eficiente por meio de builds, testes e entregas automatizadas.

Isso reduz erros, acelera a entrega e aumenta a confiabilidade do processo.
