# Formação DevOps Fundamentals - DIO

Este repositório foi criado como parte do desafio prático da Formação DevOps Fundamentals da DIO.

O objetivo é consolidar os principais conceitos aprendidos durante a formação, utilizando Git e GitHub para versionamento, documentação técnica e construção de portfólio profissional.

## Objetivo do Projeto

Documentar os fundamentos de DevOps e versionamento de código, reunindo anotações, comandos Git, conceitos de CI/CD, DevSecOps, Terraform e boas práticas de colaboração em projetos de software.

Resumo dos Conceitos
DevOps

DevOps é um conjunto de práticas, cultura e ferramentas que promove a integração entre desenvolvimento e operações. Seu objetivo é acelerar a entrega de software com mais qualidade, automação, colaboração e estabilidade.

Git

Git é um sistema de controle de versão distribuído, utilizado para registrar alterações no código-fonte, permitir colaboração entre desenvolvedores e manter histórico das mudanças realizadas no projeto.

GitHub

GitHub é uma plataforma de hospedagem de repositórios Git, usada para armazenar código, documentar projetos, colaborar com outras pessoas e construir portfólio técnico.

CI/CD

CI/CD significa Integração Contínua e Entrega Contínua. O objetivo é automatizar etapas como build, testes, validações e entrega de software, tornando o processo mais rápido, seguro e confiável.

DevSecOps

DevSecOps integra segurança ao ciclo DevOps, garantindo que práticas de segurança e conformidade sejam aplicadas desde o planejamento até a operação do software em produção.

Terraform

Terraform é uma ferramenta de Infraestrutura como Código criada pela HashiCorp. Ela permite criar, alterar e versionar recursos de infraestrutura por meio de arquivos de configuração.
